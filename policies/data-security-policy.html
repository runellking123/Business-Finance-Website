1. Purpose

This policy establishes standards for the classification, handling, storage, and protection of university data. The purpose is to ensure the confidentiality, integrity, and availability of institutional data while maintaining compliance with applicable federal and state regulations, including FERPA, GLBA, HIPAA, and Texas data protection laws.

(Reference: SACSCOC Standard 12.4 - Student Records; FERPA 34 CFR Part 99; Texas Business and Commerce Code Chapter 521)

2. Scope

This policy applies to:

  • All data created, collected, stored, or processed by the University
  • All employees, students, contractors, and vendors who access university data
  • All systems and devices used to store or process university data
  • Both electronic and physical data formats
  • Data stored on-premises, in cloud services, or on portable devices

3. Definitions

Personally Identifiable Information (PII): Information that can identify a specific individual, including name, SSN, driver's license number, financial account numbers, or biometric data.

Education Records: Records directly related to a student maintained by the University (protected under FERPA).

Protected Health Information (PHI): Health information linked to an identifiable individual (protected under HIPAA).

Data Breach: Unauthorized acquisition of computerized data that compromises the security or confidentiality of sensitive personal information.

Encryption: The process of encoding data so it can only be accessed by authorized parties.

Data Custodian: Individual responsible for implementing security controls for specific data sets.

Data Owner: Individual or department with accountability for specific data classifications.

4. Data Classification

All university data must be classified into one of four categories:

Classification Description Examples
Restricted Highly sensitive data requiring maximum protection; unauthorized disclosure could cause severe harm SSNs, financial account numbers, health records, authentication credentials, research subject data
Confidential Sensitive data requiring significant protection; disclosure limited to those with legitimate need Student education records (FERPA), employee personnel records, donor information, legal documents
Internal Data intended for internal university use; not for public disclosure Internal memos, draft documents, meeting minutes, operational data
Public Data approved for public access; no restrictions on disclosure Published catalogs, press releases, public website content, directory information (if not restricted)

5. Data Handling Requirements

5.1 Restricted Data

  • Must be encrypted at rest and in transit
  • Access limited to specifically authorized individuals
  • Must not be stored on personal devices without encryption and approval
  • Must not be transmitted via unencrypted email
  • Requires secure disposal methods (shredding, secure deletion)
  • Access must be logged and auditable
  • Requires annual access review

5.2 Confidential Data

  • Encryption required for storage on portable devices
  • Access granted on need-to-know basis
  • Should not be shared externally without authorization
  • Requires secure disposal
  • Storage in approved systems only

5.3 Internal Data

  • Access limited to university community
  • Standard security controls apply
  • Should not be shared publicly without review

5.4 Public Data

  • No special handling requirements
  • Accuracy and timeliness should be maintained

6. Security Controls

6.1 Technical Controls

  • Firewalls and intrusion detection systems on all networks
  • Antivirus and anti-malware software on all devices
  • Encryption for sensitive data at rest and in transit
  • Multi-factor authentication for access to sensitive systems
  • Regular security patches and updates
  • Secure configuration standards for all systems
  • Regular vulnerability scanning and penetration testing

6.2 Administrative Controls

  • Background checks for employees with access to sensitive data
  • Security awareness training for all users
  • Access provisioning and deprovisioning procedures
  • Regular access reviews and recertification
  • Incident response procedures
  • Vendor security assessments

6.3 Physical Controls

  • Secure access to data centers and server rooms
  • Environmental controls (fire suppression, climate control)
  • Secure disposal of physical media
  • Clean desk policy for sensitive documents
  • Visitor escort requirements in sensitive areas

7. Data Breach Response

7.1 Incident Detection and Reporting

  • All suspected data breaches must be reported immediately to IT Security
  • Do not attempt to investigate or remediate without IT involvement
  • Preserve evidence and document all known facts
  • Complete Data Security Incident Report Form

7.2 Incident Response Process

  1. Containment: Isolate affected systems to prevent further unauthorized access
  2. Investigation: Determine scope, cause, and data affected
  3. Notification: Notify affected individuals and authorities as required by law
  4. Remediation: Implement fixes to prevent recurrence
  5. Documentation: Complete incident report and lessons learned

7.3 Notification Requirements

Texas law (Business and Commerce Code 521.053) requires notification when:

  • Sensitive personal information was acquired by unauthorized person
  • Notification must be made without unreasonable delay
  • Attorney General must be notified if more than 250 Texas residents affected

(Reference: Texas Business and Commerce Code Chapter 521)

8. Regulatory Compliance

8.1 FERPA Compliance

  • Education records protected from unauthorized disclosure
  • Students have right to access and amend their records
  • Annual notification of FERPA rights required
  • Directory information may be released unless student opts out

8.2 GLBA Compliance

  • Financial information of students protected
  • Information security program required
  • Risk assessment conducted annually

8.3 PCI DSS Compliance

  • Credit card data handled per PCI standards
  • Payment card numbers not stored after authorization
  • Annual compliance validation required

9. Data Retention and Disposal

9.1 Retention

  • Data retained per university Records Retention Schedule
  • Legal holds may extend retention requirements
  • Data owners responsible for managing retention

9.2 Secure Disposal

  • Paper documents with sensitive data: cross-cut shredding
  • Electronic media: secure wiping or physical destruction
  • Certificate of destruction required for restricted data
  • Coordinate disposal with IT for electronic devices

10. Third-Party Data Sharing

  • Contracts required for vendors accessing university data
  • Data protection requirements included in all agreements
  • Security assessments required for vendors handling sensitive data
  • Business Associate Agreements required for PHI
  • Cloud services must meet university security requirements

11. Training Requirements

  • All employees: Annual security awareness training
  • Employees with access to restricted data: Additional data handling training
  • IT staff: Technical security training
  • New employees: Training within 30 days of hire

12. Roles and Responsibilities

12.1 Chief Information Officer

  • Overall responsibility for data security program
  • Approves security policies and standards
  • Reports security status to university leadership

12.2 IT Security

  • Implements and monitors security controls
  • Investigates security incidents
  • Conducts security assessments
  • Provides security awareness training

12.3 Data Owners

  • Classify data under their responsibility
  • Authorize access to their data
  • Ensure compliance with handling requirements

12.4 All Users

  • Protect data in their custody
  • Report security incidents
  • Complete required training
  • Follow data handling procedures

13. Compliance and Review

This policy is reviewed annually by IT Security and updated as needed to address new threats, technologies, and regulatory requirements. Compliance is monitored through audits, assessments, and incident tracking.

14. SACSCOC Standards Referenced

  • Standard 12.4 (Student Records): The institution protects student rights to privacy and provides access to educational records in compliance with FERPA.
  • Standard 13.7 (Physical Resources): The institution maintains technology infrastructure and information security adequate to support its mission.
